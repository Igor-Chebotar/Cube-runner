## Что это

3D-раннер на Unity. Куб бежит вперёд по бесконечной дорожке, на пути — препятствия двух видов и бонусы. Задача — уклоняться, прыгать и выживать.

## Как устроено

Две сцены: `MainMenu` (стартовый экран) и `Game` (сам геймплей). Автосборка всего проекта — через Editor-скрипт `Runner -> Setup Full Project`.

Основные скрипты:
- `GameManager` — синглтон, рулит состоянием: пауза, смерть, переход между сценами
- `PlayerController` — движение, прыжки, HP, бонусы, визуальный отклик (корутины + Animator)
- `ObstacleSpawner` — спавн препятствий и бонусов впереди игрока, удаление позади, а также динамическая генерация земли чтобы пол не кончался

## Управление

A/D или стрелки — лево/право, пробел — прыжок (можно двойной), ESC — пауза

## Данные объектов

Характеристики хранятся в ScriptableObject (`ObstacleData`, `BonusData`), как требуют общие требования — чтобы не дублировать данные в памяти при спавне одинаковых объектов.

## Бонусы

Три типа: лечение (зелёный), ускорение (жёлтый), неуязвимость (голубой). Каждый — отдельный SO с настраиваемыми параметрами.

## UI

- Главное меню: название игры, имя разработчика, рекорд, кнопки «Играть» и «Выход»
- HUD: полоса HP, текущий счёт, рекорд
- Пауза (ESC): затемнение + кнопки «Продолжить» / «В меню»

## Сохранение

Рекорд — через `PlayerPrefs`. Сохраняется при смерти если текущий счёт больше предыдущего.

## Анимации

Используется Unity Animator — контроллер с состояниями Run, Hit, BonusPickup и параметрами (IsGrounded, Speed, Invincible, SpeedBoosted). Плюс перекрашивание материала через корутины как дополнительный визуальный отклик:
- Урон — мигание красным
- Хил — зелёная вспышка
- Неуязвимость — голубой цвет на время действия
- Ускорение — жёлтый цвет на время действия
